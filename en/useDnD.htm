Drag and Drop
Dragging and Dropping onto Program Buttons
When files are dragged and dropped onto a button with a program, batch file, or script (such as VBScript) that accepts command-line arguments, the registered item opens the dropped files.
When files are dropped onto a document item, both the registered item and the dropped files are opened using the program associated with the file type.
Dragging and Dropping onto Folder Buttons
When files are dropped onto a folder button, the dropped files are copied into that folder.
Holding down the Shift key while dragging causes the files to be moved instead of copied.
The default action when files are dropped (copy or move) can be changed in the When files are dropped onto a folder button setting on the Folders tab of the Options dialog.
If files are dragged and dropped using the right mouse button, a menu is displayed to select the desired operation.

Update: Only files newer than those already in the destination folder are copied and overwrite existing files without confirmation. (Files that do not exist in the destination folder are always copied.)
Create Shortcut: Creates a shortcut with the same name as the file in the destination folder.
Dragging and Dropping onto the Recycle Bin
When files are dropped onto a Recycle Bin button, the files are moved to the Recycle Bin. No confirmation message is displayed.
Holding down the Shift key while dragging permanently deletes the dropped files instead of moving them to the Recycle Bin.
If files are dragged and dropped using the right mouse button, a menu is displayed to choose between Move to Recycle Bin and Delete Completely.
OLE Drag-and-drop
Dragging a registered item using the right mouse button allows it to be dropped onto program buttons or applications that support OLE drag and drop. This allows, for example, opening document items with a program other than the default associated application, or opening URL items with a browser other than the default browser.
OLE drag and drop is available only for program items, document items, regular folders, and URL items. UWP apps (Store apps), special folders, special items, and Control Panel items do not support OLE drag and drop.
When files are moved by dragging and dropping them into Explorer, the information of the items registered in CLaunch is not updated.
The action used to start OLE drag and drop with the left mouse button can be configured in the Drag-and-drop setting on the Mouse tab of the Options dialog.
