Menu Options
You can change the behavior of menus by specifying Menu Options in the fourth input field of the Item Properties dialog for submenus and folder items.
To specify multiple words, separate them with spaces.
Common options for Submenu and Folder Contents Menu

/w:Limits the width of menu items. It takes precedence over the Column Width setting in the Menus tab of the Options dialog.
Format : /w[column width]
/r:Limits the number of rows of the menu. It takes precedence over the Number of Rows setting in the Menus tab of the Options dialog.
Format : /r[number of rows]
/c:Specifies the number of columns of the menu. It takes precedence over the Number of Columns setting in the Menus tab of the Options dialog.
Format : /c[number of columns]
/t:Tooltips will be displayed when the cursor is over the menu items. It takes precedence over the Show tooltips setting in the Menus tab of the Options dialog.
/is:Specifies the display size of menu icons. It takes precedence over the Icon size setting in the Menus tab of the Options dialog.
Format : /is[icon size]
/il:Uses the icons for buttons. It takes precedence over the Use large icons setting in the Menus tab of the Options dialog.
/p:Specifies the coordinates to display the menu.
Format : /p[X coordinate]:[Y coordinate]

Examples of menu options:
/r10 /c3Displays the menu in 10 rows and 3 columns.
/il /is32Uses the icons for buttons and displays them by 32 x 32 pixels.
/p400:300The menu will be displayed to coordinates of X=400 and Y=300.
Options exclusive to the Folder Contents Menu
In addition to the common options listed above, the following dedicated options and filter strings are available in the Folder Contents Menu.

By specifying a filter string, you can display only files whose names match a specific pattern.
The wildcards "*" and "?" can be specified in the filter string. "*" will be replaced with any string of zero or more characters, and "?" will be replaced with any single character.
If no sort option is specified, the files will be sorted in ascending order by file name.

/max:Limits the number of files to display. It takes precedence over the Number of Items setting in the Menus tab of the Options dialog.
Format : /max[number of files]
/in:The icons will not be displayed.
/h:The hidden files will be displayed. (The system files will also be displayed in Windows 7 or later.)
/f:Specifies the filter operation by combination with the following characters.
i:The files specified by the filter strings will be hidden.
d:The folders will be included to the filtering target.
/s:Specifies the sorting operation by combination with the following characters. (Only one of e, s, u, c, a can be specified.)
e:Sorting by file extension.
s:Sorting by file size.
u:Sorting by timestamp of update.
c:Sorting by timestamp of creation.
a:Sorting by timestamp of access.
r:Sorting in descending order.
d:Sorting without dividing the folders and files.
n:Inhibits the sorting.
/i:Child menus will inherit the filter strings and sort options.

Examples of menu options:
/h /in /ssrThe menu will be displayed in descending order of file size including hidden files without icons.
*.txt *.htmlOnly files that have extensions ".txt" or ".html" will be displayed.
*.dat *.tmp /fiThe files that have extensions ".dat" and ".tmp" will be hidden.
abc???.jpg /surOnly ".jpg" files whose names start with "abc" and have six characters after it will be displayed in descending order of last modified time.
