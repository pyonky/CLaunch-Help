Advanced Options for Already Running Program dialog
This dialog is displayed when you click the Advanced Options button for the Action when the program is already running setting in the Item Properties dialog of an item that has a program or document registered.
Action when the program is already running
Selects the behavior when you try to launch a program again after it has already been launched.
The actual behavior depends on the program, so it may not work as expected.
If you have problems launching the program, set it to Default.
  • Default
    Launches the same program again.
  • Activate
    Activates the window of the running program.
  • Close
    Closes the window of the running program.
  • Do Nothing
    Does nothing.
    By specifying Filtering conditions for target windows, you can configure it to launch the program only when no window that matches the specified conditions exists.

Show hidden windows (when Activate is selected)
If the target window to be activated is hidden, it will be made visible.
This is useful for applications that minimize to the task tray, but windows that were intentionally hidden may also be shown.
Therefore, when enabling this option, set Filtering conditions for target windows as needed.

Terminate the process forcefully (when Close is selected)
Forcibly terminates windows that could not be closed by the normal procedure.
Force termination may cause data corruption or data loss. Use at your own risk.

Limit the target to processes launched from this item
Limits the target to windows opened by the item that this dialog is configured for.
If disabled, all windows are targeted, including processes launched from Explorer, etc.
If the registered program is launched as administrator, this option is ignored.

Change the target to the specified program
Changes the target from the program launched by the item that this dialog is configured for to a different program.
This is for special application configurations where a program launched first starts another program as a secondary launch.

Filtering conditions for target windows
Configure the filtering conditions for target windows.
  • Target all windows created by this program
    No filtering is performed.
  • Limit to windows that meet the following conditions
    Windows that do not match the specified conditions for window title and window class are excluded from the target.
  • Exclude windows that meet the following conditions
    Windows that match the specified conditions for window title and window class are excluded from the target.

When a blank row in the list is selected, entering a condition in Window Title or Window Class will automatically register it as a filtering condition.
Click the Existing Windows button to show a list of running applications. Select the target application and click OK to retrieve its information.
If you cannot find the target application in the list, click the Window Finder button and then click anywhere in the target application's window to automatically select the corresponding item in the list.
If multiple conditions are set, only windows that match all conditions will be targeted, so delete unnecessary conditions and leave them blank.
Window Title is evaluated by partial match, and Window Class is evaluated by prefix match.
To add another condition, reselect a blank row and then enter new conditions.
