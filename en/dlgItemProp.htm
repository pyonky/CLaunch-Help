Item Properties dialog
Displayed when you select Properties from the Item Menu, or when you right-click an item while holding down the Alt key.
Name
The text displayed on the button or menu item.
Target
The target file name of the registered item.
Enter the path of a file or folder, or a URL.
You can use relative paths from the CLaunch installation folder and environment variables.
Parameters
Specifies command line parameters when launching a program.
If you set <clipboard> (clipboard replacement tag) in the parameters of program, document, or URL items, that part will be replaced with the string in the clipboard and passed to the command line of the program to be launched.
For example, if you create an item with the URL "https://www.google.co.jp" and the parameters "/search?q=<clipboard>", then launch it while some text is copied to the clipboard, you can search Google using the clipboard text as a keyword.
The parameters for folder items are meaningful only when the Use an alternative file manager setting in the Folders tab of the Options dialog is enabled, and the parameter string is passed to the file manager's command line.
For special folders, specify strings such as "shell:Desktop" (Desktop) or "::{450D8FBA-AD25-11D0-98A8-0800361B1103}" (My Documents). However, the file manager you use must support these strings that represent virtual folders.
Start in (except submenus and folders)
Specifies the current directory when launching a program.
You can use relative paths from the CLaunch installation folder and environment variables.
Menu Options (submenus and folders only)
Specifies an option string that controls the display behavior of submenus, the Folder Contents Menu, and menu-type special items.
Menu Options
Run
Specifies the window state when launching a program.
Priority
Specifies the priority when launching a program.
This setting is available only for program or document items.
If you have problems launching a program, set it to Normal.
If you overuse Realtime or High, the overall system performance will degrade.
Privilege Level (Windows Vista or later)
Specifies the privilege level (the permission level of an application in a UAC environment) when launching a program.
By default, the launched application's privilege level inherits CLaunch's privilege level.
Therefore, the meaning of this setting changes as follows depending on the privilege level used to start CLaunch.
When CLaunch is running as a standard user
The default becomes Run as Standard User, and program items can select Run as Administrator.
When CLaunch is running as administrator
The default becomes Run as Administrator, and program and document items can select Run as Standard User.
Tooltip
Specifies the text displayed when the cursor stays over the button or menu item.
Shortcut Key
Sets the keyboard operation to launch an item when the page or submenu that contains the item is displayed.
After typing the key you want to assign, use the spin button on the right to set how many consecutive presses are required for it to be accepted as the shortcut key.
Keys assigned to basic operations cannot be specified.
Shortcut keys work even for items that become hidden outside the visible area when you reduce the CLaunch window size.
Hotkey
Sets the keyboard operation to launch an item when the CLaunch window is not displayed.
After typing the key you want to assign, use the spin button on the right to set how many consecutive presses are required for it to be accepted as the hotkey.
Keys assigned to Hotkey for displaying CLaunch cannot be specified.
If you register a key that is already assigned as a hotkey for another item, the previously registered hotkey will be cleared.
Avoid using a single alphanumeric key as a hotkey; use a combination with [Ctrl], [Alt], or the [Windows] key. (The program will accept registration even for a single character key unconditionally.)
Hotkeys may conflict with other applications or resident software, so not all combinations can be used.
Mouse Launch
Sets mouse operations to launch an item when the CLaunch window is not displayed.
Clicking the Assign button displays the Direct Mouse Launch dialog.

Chain Launch Items
Sets the items to launch together when you launch this item.
Clicking it displays the Auto-start Items dialog.
In the Auto-start Items dialog, click the Add button to show page names in a drop-down list, then select the page or submenu that contains the item you want.
Double-click a cell in the Item column of the added row to show items in the selected page in a drop-down list, then select the item you want.
Double-click a cell in the Delay column of a row that has a launch item to set the wait time before additional items are launched after the first item starts.
If you set a negative value, the additional item will be launched before the main item.
You can reorder the list with the Up and Down buttons, but items are not guaranteed to be launched in the displayed order.
Change Icon
Specifies the icon and overlay icon displayed on the button or menu item.
Clicking it displays the Icon Settings dialog.
You can specify any .exe, .dll, .ico, or .icl file as an icon file.
The icon displayed in the preview of Menus is shared by submenus and the search result list in the search window, and by default a 16 x 16 pixel icon is displayed.
If the Separate designs per View Mode setting in the Design 2 tab of the Options dialog is enabled, icon images obtained for View Mode 2 are used.
If you check Show Overlay Icon and specify an icon file, an additional icon will be displayed over the main icon.
Overlay Icons
Don't update icon
When you click OK, it does not retrieve the icon data again and uses the cached icon images. Also, it is excluded from being updated when you run Updating Icon Images.
Show Shell menu on right-click
When you right-click an item, the Shell context menu will be displayed instead of the CLaunch item menu.
If you enable this setting, right-click while holding down the Shift key to show the item menu.

Action when the program is already running (program or document)
Selects the behavior when you try to launch a program again after it has already been launched.
The actual behavior depends on the program, so it may not work as expected.
If you have problems launching the program, set it to Default.
  • Default
    Launches the same program again.
  • Activate
    Activates the window of the running program.
  • Close
    Closes the window of the running program.
  • Do Nothing
    Does nothing.
    By specifying Filtering conditions for target windows in the advanced options, you can configure it to launch the program only when no window that matches the specified conditions exists.
Clicking the Advanced Options button displays the Advanced Options for Already Running Program dialog.
Auto-unhook Settings (program or document)
Displays the Auto-unhook Settings dialog and configures settings to automatically perform Unhook and resume according to the start and exit of the program.

Methods to open folders (folder item)
Selects how to open the folder.
  • Open
    Opens the folder in the system default folder window.
  • Open in Explorer
    Opens the folder in Explorer with the navigation pane visible.
  • Open as Menu
    Opens the folder contents as a Folder Contents Menu.
Auto-open menu on mouseover (submenu or folder item)
Automatically opens the menu just by moving the cursor over a submenu button, or a folder button whose method is set to Open as Menu.
An auto-opened menu closes automatically when you move the cursor over another button.
The wait time before auto-opening starts can be changed in Menus of the Options dialog.
Display Position
You can select where the menu will appear when it is auto-opened.

Additional Options (special item: Recent Items)
Launch last used item
When you click the item, it launches the last launched item directly without showing the history menu.
The icon and tooltip of this item will reflect the last launched item.
Show history menu on right-click
When you right-click the item with the above option enabled, the history menu will be displayed instead of the item menu.
To display the item menu with this option enabled, right-click while holding down the Shift key.
History Management
Displays the History Management dialog to configure history recording behavior.

Target Menu (special item: Last Displayed Menu)
Selects the type of menu to redisplay.
  • Submenus
    Redisplays the last displayed submenu.
    Folders opened as menus are not included.
  • Folder Contents
    Opens the last folder that was opened as a menu again.
    Submenus are not included.
  • Both
    Redisplays the last displayed menu, whether it is a submenu or a Folder Contents Menu.
Additional Options (special item: Clipboard Folder Menu)
Open parent folder even if the target is a folder
If a folder path is copied to the clipboard, normally that folder is opened as a menu. If you enable this option, the parent folder (one level up) will be opened instead.
If a file path is copied, the behavior does not change.

Additional Options (special item: Suspend Screen Saver)
Resume when the cursor leaves the edge
An option for assigning the suspend screen saver function to Edge Functions.
After you move the cursor to the screen edge to suspend the screen saver, the suspension will be released when the cursor leaves the edge.

Additional Options (power control items)
Ask for Confirmation
Displays a confirmation message before starting shutdown processing.
Force Shutdown
Performs shutdown without waiting for running applications to respond.
Because it does not wait for applications to respond, problems such as loss of unsaved data may occur. Use at your own risk.

Plugin Item Options (plugin item)
Item Properties
Displays a dialog to configure settings for the plugin item.
Plugin Options
Displays a dialog to configure general settings for the plugin.
