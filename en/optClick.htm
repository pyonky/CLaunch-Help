Clicks (Options dialog)
Click Functions
Allows you to show CLaunch or launch registered items when click-based operations are performed.

Click the Single, Double, Triple, Wheel, or Press & Hold buttons at the top to display the list of corresponding click operations.
Click the [+] next to an operation name to expand its effective areas. Select the checkbox next to an area to allow CLaunch to be shown by that operation.
If you want CLaunch to be shown regardless of location, check Anywhere.
When you click a checkbox, the Initial Page dialog appears. Select the page you want to display first.
If you choose Last Displayed Page, the page that was visible before CLaunch was hidden will be displayed.
To change the initial page, uncheck the option once and then check it again to reopen the Initial Page dialog.
If the desired operation is already assigned to launching an item, uncheck it once to remove the current assignment, then check it again.
Mouse direct launch for individual items can be configured from Mouse Launch in the Item Properties dialog.
Hide CLaunch when the cursor leaves the window
After CLaunch is shown by a click function, it will automatically be hidden when the cursor leaves the window.
Emulate a delayed single click
Changes the behavior when a function is assigned to the double-click or triple-click of the wheel button, X1 button, or X2 button, and nothing is assigned to single click.
When enabled, a single click does not immediately trigger the system’s default action. Instead, the single-click action occurs only after the Windows double-click time (500 ms by default) has elapsed without a second click. (The double-click time can be changed in Mouse Properties in Control Panel.)
For example, if showing CLaunch is assigned to the double-click of X1 button:
If this option is disabled, double-clicking X1 button in a browser will execute “Back” on the first click and then show CLaunch on the second click.
If this option is enabled, clicking once and then clicking again within 500 ms will show CLaunch. If no second click occurs within 500 ms, the browser’s “Back” action will be executed.
Because this option significantly affects usability, it does not apply to the left and right mouse buttons.
Wait for the double/triple-click interval
Allows different functions to be assigned to single, double, and triple clicks of the same mouse button.
If disabled and you perform a double-click, the single-click assignment is triggered on the first click, and the double-click assignment is triggered on the second click.
If enabled, after the first click, CLaunch waits up to 500 ms (the double-click time). If a second click occurs within that time, the double-click assignment is executed. If no second click occurs, the single-click assignment is executed.
The same behavior applies to triple-clicks.
This option applies to all mouse buttons.
Cancel press-and-hold when the cursor is moved
If the cursor moves after pressing a mouse button, press-and-hold detection is suspended until the button is released and pressed again, preventing unintended activation.
If you want to show CLaunch by press-and-hold in order to drag and drop files, uncheck this option.
Use low-level mouse hook
Enable this option if performance degradation occurs in games.
It may also resolve issues where click functions or Circle functions do not work on specific windows.
If launched from ClAdmin, this option is forcibly enabled.
On operating systems earlier than Windows 2000, this option is disabled.
Press-and-hold delay
Specifies the waiting time between pressing a mouse button and executing the function assigned to a press-and-hold operation.
