ClAdmin
In environments from Windows Vista onward, due to UAC (User Account Control), when a program running with administrator privileges (such as application installers or Task Manager) is active, CLaunch cannot be displayed using mouse operations or hotkeys, and registered items cannot be launched directly.
This restriction can be avoided by running CLaunch itself with administrator privileges. However, in that case, CLaunch will no longer be able to receive files via drag and drop from programs running with standard user privileges, such as Explorer.
ClAdmin is a startup module designed to resolve these issues simultaneously.

When ClAdmin is launched, CLaunch is started at the same time.
When launching ClAdmin, you can specify the same command line parameters as CLaunch.
If ClAdmin is launched manually, a UAC confirmation dialog will be displayed. (The message asks whether to allow changes to this computer, but no actual changes are made.)
If Create a Startup Shortcut and Start up with ClAdmin are selected during setup, ClAdmin will start automatically when Windows starts. In this case, no confirmation dialog will be displayed.
